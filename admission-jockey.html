Project Detail

The "Admission Jockey Website with Admin Panel" is a comprehensive platform developed using the MERN (MongoDB, Express.js, React.js, Node.js) stack. It is designed to serve both students seeking admission and administrators managing the application. The website offers a wide array of features to streamline the admission process and ensures efficient administration.

Key Features:
For Student:
  • College and University Search: Aspiring students can search for colleges and universities using various filters, such as type (public/private), name, state, city, and available courses. The search results are generated dynamically, and each query creates a unique URL for easy sharing.
  • College Predictor Function: A predictive feature assists students by generating a list of colleges based on their responses to specific questions. This personalized approach simplifies the college selection process.
  • College Comparison: Students can compare multiple colleges and universities side by side, considering factors such as location, programs, tuition fees, and admission requirements.
  • Individual College Details: Detailed pages provide comprehensive information about each institution. Students can access data about the college's history, faculty, campus facilities, and more.
  • Query and Callback Request: Aspiring candidates can raise queries or request a callback for assistance. A dedicated section allows users to submit questions or contact information, enabling the admission team to provide guidance and support.
  • Engaging Design: Create an attractive and interactive user interface.
For Admins:
  • Dashboard: The admin panel features a dashboard that provides an overview of user activities, queries, and website statistics.
  • User Management: Administrators can manage user accounts, view user data, and track user interactions with the website.
  • Content Management: Admins have control over the content displayed on the website. They can add, edit, or remove information about colleges and universities.
  • Query Handling: The admin panel includes a system for managing user queries and callback requests, streamlining the support process.
  • Role-Based Access Control: Administrators can define and manage user roles with role-based access control. Roles can include Editors, Callers, Super Admins, Project Managers, and Team Leaders. Different roles have specific permissions and access to the dashboard, ensuring efficient collaboration and task management.
  • Analytics: Detailed analytics and reporting tools assist administrators in assessing website performance and user engagement.
